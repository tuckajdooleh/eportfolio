Spotify Group Listening (Using Selenium)

By: Tucker Dooley

I made this project so that when I'm hanging with friends we can all easily add/remove songs from the hosts spotify queue without having to make the host manually enter or remove peoples songs all the time.

I realize that this project could have been done using spotify's API, but I had been enjoying working with browser automation, so I decided to use Selenium. I do intend to make an API version of this project in the future, because the browser automation apporach I used breaks everytime Spotify updates their website.

This project consists of 2 applications. The first is the node js / socket IO application. This just serves the request HTML file where anyone on the current network can visit and enter a song. Once a user enters a song and presses submit, the server adds that text to a local text file. It will also remove songs when users press remove. This project trusts that no one in the party will sabotage the queue by removing everyones song requests.

The second application is the Python Selenium application. The python program uses Chromedriver to automate Spotify's webplayer. On starting the program it pulls up the login page for the webplayer, it then automatically inputs my username and password and pressed login. It then checks the textfile for the song at the top of the queue, if there is none, it waits 2 seconds and repeats. If there is a song it appends the name of the song to a search url, it then presses tab 25 times and enter once, to select the first song that comes up from the search. This application trusts that the users type the songs in a way that their desired song comes up first. I eventually plan to user Spotify's API to add a search menu for users to use to physically click on their exact song.
